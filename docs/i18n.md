Internationalization (i18n)

Overview
- JSON translation files live in `src/i18n/locales/<lang>.json` (e.g., `en.json`, `ro.json`).
- A lightweight helper in `src/i18n/index.ts` provides `t(key)`, `setLocale(lang)`, and `translateDocument(document)`.
- Electron main uses `t(...)` for window titles and menu labels.
- Preload exposes `window.i18n` to renderer pages and auto-translates elements with `data-i18n` attributes.

Renderer Usage
- Add `data-i18n="your.key"` to any element and the preload will set its `textContent` when the DOM loads or when language changes.
- Optionally call `window.i18n.translatePage()` after dynamic DOM changes.
- Change language from renderer: `window.i18n.setLocale('ro')`.

Main Process
- Main initializes locale from `app.getLocale()` and builds menus and window titles using `t(...)`.
- Language menu is generated from available JSON files and updates all windows on change.

Adding a Language
- Copy `src/i18n/locales/en.json` to a new file (e.g., `src/i18n/locales/fr.json`) and translate values.
- Restart the app; the new language appears under the Language menu.

Consistency Check
- Run `npm run i18n:check` to detect missing or extra keys compared to `en.json`.
